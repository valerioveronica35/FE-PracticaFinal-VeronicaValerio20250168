ExplicaciĂłn de los comandos

ConfiguraciĂłn bĂˇsica

* `hostname`: cambia el nombre del equipo
* `ip domain-name`: necesario para generar claves SSH
* `username`: crea usuarios locales
* `line vty`: acceso remoto
* `login local`: usa los usuarios creados
* `transport input ssh`: solo permite SSH

---

Subinterfaces (Router-on-a-Stick)

* `interface gi0/0.300`: crea subinterfaz
* `encapsulation dot1q 300`: asigna VLAN
* `ip address`: IP del gateway real

Esto permite que un router maneje varias VLANs.

---

DHCP Relay

* `ip helper-address 192.168.100.10`

EnvĂ­a las solicitudes DHCP al servidor (R2).
Sin esto, los clientes no reciben IP.

---

HSRP

* `standby X ip`: gateway virtual
* `standby priority`: define el activo
* `preempt`: recupera el rol si vuelve

R4 tiene mayor prioridad, por eso es el gateway principal.

---

Rutas estĂˇticas

* `ip route red mascara siguiente-salto`

Sirve para alcanzar otras redes.

* `0.0.0.0 0.0.0.0`: ruta por defecto

---

DHCP (R2)

* `ip dhcp pool`: crea pool
* `network`: red a asignar
* `default-router`: gateway
* `dns-server`: DNS
* `excluded-address`: evita asignar IPs reservadas

---
VLANs (Switch)

* `vlan X`: crea VLAN
* `name`: nombre

---

Trunk

* `switchport mode trunk`: habilita trunk
* `switchport trunk allowed vlan`: VLANs permitidas
* `native vlan`: VLAN sin etiqueta

---

EtherChannel

* `channel-group 1 mode active`: LACP

Une varios enlaces en uno lĂłgico.

---

STP

* `spanning-tree vlan X root primary`

Define el switch principal para evitar loops.

---
